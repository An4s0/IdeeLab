# Brand Assets – IdeeLab

This folder contains all branding assets and identity resources for the **IdeeLab** project. These assets are essential for maintaining visual consistency across all platforms, including the website, social media, documentation, and marketing materials.

---

## colors/

Contains the official color palette of the brand.

- `palette.json`: JSON file with all primary, secondary, and neutral colors in HEX/RGB format.
- `palette.png`: A visual reference for the full color palette.
- `palette.psd`: Photoshop file for designers to use the palette in design tools.

---

## logos/

Contains the primary and alternate versions of the IdeeLab logo.

### png/
- `logo.png`: Standard logo on a light background.
- `logo.white.png`: Logo in white, for use on dark backgrounds.

### svg/
- `logo.svg`: Scalable version of the main logo.
- `logo.white.svg`: White logo (for dark backgrounds), optimized for web use.

> Use **SVG** logos for better scalability and quality on all screen sizes whenever possible.

---

## typography/

Typography files and references used in the project branding.

### fonts/
- `FiraCode-Regular.ttf` – Primary typeface used for body text and other elements.
- Includes all font weights (Light, Regular, Medium, SemiBold, Bold).

### font-sample.png
A visual guide showing how the font looks in different weights and sizes.

---

## Usage Guidelines

- **Logos**: Do not distort, recolor, or alter the logo files in any way. Always use the appropriate version (light/dark) depending on background.
- **Colors**: Use colors from the official palette only. Keep consistent contrast and brand tone.
- **Typography**: Use "Fira Code" for headings, buttons, and any code-related UI elements.
